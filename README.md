### Задание 1

Задание 1
На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме.


Ответ 

Master-Slave Ведущий-ведомый - Архитектура с одним направлением данных. Единственный источник истины мастер, данные от которого тиражируются на подчиненные узлы, Разгрузка master чтение распределяется по slave-серверам, что повышает общую производительность, резервное копирование, аналитика и отчёты, простота настройки, масштабируемость чтения: можно добавлять новые slave для увеличения пропускной способности на чтение. Master-Master Ведущий-ведущий - Архитектура с двунаправленной репликацией. Несколько узлов равноправны, могут принимать запись и обмениваться данными между собой, основные плюсы высокая отказоустойчивость, балансировка нагрузки, географическое распределение.

Ключевые отличия в том что Master‑Slave подходит для сценариев, где Преобладают операции чтения, для простоты и надёжности и допустима некоторая задержка репликации, а Master‑Master выбирают, тогда когда критична высокая доступность, нужно распределить нагрузку на запись между серверами, Система географически распределена.
